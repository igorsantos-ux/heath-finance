# OBJETIVO DO PROJETO: "ClinicaFin SaaS"
Desenvolver uma aplicação web para gestão financeira de clínicas médicas. O sistema deve sair da lógica de "apenas visualização" (Power BI) para um sistema de gestão ativo, onde o usuário insere metas, gerencia estoques e visualiza projeções.

# 🧠 SKILLS E DIRETRIZES TÉCNICAS (Obrigatório seguir)
Para este projeto, ative suas habilidades de:
1.  **Clean Architecture:** Separação estrita de responsabilidades. O Backend é a única fonte da verdade. O Frontend apenas renderiza.
2.  **Component-Driven Development (CDD):** Crie componentes pequenos, reutilizáveis e isolados no React.
3.  **Docker-First:** O ambiente de desenvolvimento deve ser encapsulado em containers para garantir que funcione igual no Localhost e no Easypanel depois.
4.  **Security by Design:** Validação de dados (Zod) no backend e frontend.

# 🏗️ STACK TECNOLÓGICO
- **Estrutura de Pastas:** Raiz do projeto contendo `/frontend` e `/backend` separadamente.
- **Frontend:** React (Vite) + TypeScript + Tailwind CSS.
- **UI Library:** Shadcn/UI (para componentes como tabelas, modais, inputs) + Lucide React (ícones).
- **Gráficos:** Recharts (para reproduzir os visuais do Power BI com interatividade).
- **Backend:** Node.js (Express) com TypeScript.
- **Banco de Dados:** PostgreSQL (via Docker).
- **ORM:** Prisma (para modelagem e migrações).
- **Infra:** Docker Compose para orquestrar tudo no Localhost.

# 📊 ANÁLISE DOS MÓDULOS (Baseado nas Imagens Anexadas)
Implemente a lógica de negócio para os seguintes módulos:

1.  **Dashboard Gerencial (Visão Geral):**
    - Cards de KPI: Faturamento, Despesas, Saldo Líquido.
    - Gráfico Principal: Evolução Faturamento vs Despesas (Area Chart).
    - Gráfico Secundário: Metas (Gauge Chart/Velocímetro).

2.  **Módulo de Produtividade Médica (Complexo):**
    - **Lógica:** Tabela detalhada que calcula o "Resultado Final Produtividade".
    - **Fórmula:** Faturamento Bruto - (Impostos + Materiais + Taxas Cartão + Custo Operacional + Tecnologia).
    - Deve haver um split (divisão) claro entre o que é "Parte do Médico" e "Parte da Clínica".

3.  **Módulo de Faturamento e Fluxo de Caixa:**
    - Detalhamento por "Procedimentos" e "Tecnologias".
    - Visualização de Entradas vs Saídas mensais.
    - Projeção futura baseada na média dos últimos 3 meses.

4.  **Controle de Estoque (Sazonalidade e Curva ABC):**
    - Tabela com colunas: Código, Produto, Categoria, Estoque Atual, Estoque Mínimo, Status.
    - **Lógica de Alerta:** Se `Estoque Atual` < `Estoque Mínimo`, o status deve ficar vermelho ("Abaixo do Mínimo") e gerar um alerta na dashboard.
    - Gráfico de Pareto (Curva ABC): Quais produtos representam 80% do valor em estoque.
